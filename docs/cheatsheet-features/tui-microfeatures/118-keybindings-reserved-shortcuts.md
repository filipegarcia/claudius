# Reserved shortcut validation with platform-specific warnings

**Source:** Claude Code TUI — keybindings-vim-voice
**Status:** PARTIAL

## What it is
When the TUI's keybindings editor inspects a candidate chord, a validation layer (`src/keybindings/reservedShortcuts.ts`) hard-blocks rebinding `ctrl+c`, `ctrl+d`, and `ctrl+m` with hardcoded error strings — e.g. `"ctrl+m\nCannot be rebound - identical to Enter in terminals (both send CR)"` — softer-warns about `ctrl+z` / `ctrl+\` as terminal-reserved, and on macOS additionally blocks `cmd+c/v/x/q/w`, `cmd+tab`, and `cmd+space`. Each entry carries a friendly `reason` string (`"macOS app switcher"`, `"Unix process suspend"`) that the recorder surfaces inline next to the offending binding rather than as a generic "invalid chord" toast.

## Claudius today
Claudius has two adjacent surfaces but neither implements the terminal-reserved list. The CLI-targeted editor at `app/[workspaceId]/keybindings/page.tsx` (which writes `~/.claude/keybindings.json` via `lib/server/keybindings.ts` lines 5-40) accepts arbitrary free-form `key` strings — there is no `chord -> reserved?` check, no error string, and `KeybindingsFile` is persisted verbatim regardless of whether the chord is `ctrl+c`. The web-shortcut registry at `lib/client/shortcuts.ts` is closer in spirit: `RESERVED_BROWSER_KEYS` (line 751) and `reservedConflictFor` (line 761) already produce label strings like `"Close tab"` / `"Reload"` / `"Quit (macOS)"`, and `components/settings/ShortcutsSection.tsx` line 177 renders that label next to the recorded chord (`const reserved = isElectron ? null : reservedConflictFor(binding);`). But that list is purely *browser*-reserved (`mod+W`, `mod+T`, `mod+R`, `mod+L`, `mod+Q`, `mod+Shift+T`) — it does not cover the TUI's terminal-reserved set (`ctrl+c/d/m/z/\`) or the OS-level macOS set (`cmd+tab`, `cmd+space`, `cmd+c/v/x`), and `app/[workspaceId]/keybindings/page.tsx` does not call `reservedConflictFor` at all on the CLI-keybindings editor where the TUI rule actually applies.

## Decision
PARTIAL. The plumbing pattern already ships — `reservedConflictFor` + the inline "reserved" hint on `ShortcutsSection.tsx` — but the CLI-keybindings page accepts unsafe chords silently, and neither surface carries the TUI's terminal-reserved triplet (`ctrl+c/d/m`) or the macOS OS-reserved set with friendly `reason` strings. The follow-up is to mirror the leak's `src/keybindings/reservedShortcuts.ts` table inside `lib/server/keybindings.ts` (or a new `lib/shared/reserved-shortcuts.ts`) as a `{ chord, severity: "block" | "warn", reason }` list, then have `app/[workspaceId]/keybindings/page.tsx` show the reason next to the offending row and refuse to save block-severity rows — using the existing `reservedConflictFor` rendering convention from `ShortcutsSection.tsx` line 177 so both editors speak the same language.
