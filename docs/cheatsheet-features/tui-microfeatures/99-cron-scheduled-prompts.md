# CronCreate / CronList / CronDelete scheduled prompts

**Source:** Claude Code TUI — tools
**Status:** ALREADY_EXISTS

## What it is
Three Kairos tools that let the agent schedule a prompt to fire later — `CronCreate` takes a 5-field cron expression (recurring) or a one-shot timestamp, with `durable: true` persisting to `.claude/scheduled_tasks.json` and the default keeping the entry session-only, plus built-in jitter and a configurable max age. The `tools/ScheduleCronTool/prompt.ts` leak spells out the contract verbatim: "Schedule a prompt to run at a future time — either recurring on a cron schedule, or once at a specific time. Pass durable: true to persist to .claude/scheduled_tasks.json; otherwise session-only." Crons fire while the REPL is idle, and `CronList` / `CronDelete` round out the lifecycle.

## Claudius today
Claudius mirrors all three tools end-to-end. The agent-runtime side lives in `lib/server/session.ts` (CronCreate handler around line 2583, ScheduleWakeup around line 2651, with `durable: !/session[- ]only/i.test(text)` parsing at line 2688). The client reducer in `lib/client/use-session.ts` reconstructs the loops from the SDK tool stream: `CronCreate` lands as a pending entry keyed by `tool_use_id` (line 1993), the cron-id result re-keys it (line 2458), `CronDelete` flips `cancelled` eagerly (line 2041), and the parsed `durable` flag rides the entry through to render (line 2480). The wire shape is `SessionLoop` in `lib/shared/session-loops.ts` — `kind: "cron" | "wakeup"`, `recurring`, `durable` ("True iff CronCreate did NOT mark this session-only"), `humanSchedule`, `delaySeconds`, plus a `cancelled` flag. Browser observability is the `/schedule` page (`app/[workspaceId]/schedule/page.tsx`) backed by `app/api/schedule/session-loops/route.ts` (list) and `app/api/schedule/session-loops/cancel/route.ts` (cancel-by-prompt), with the in-session widget `components/panels/widgets/ScheduledLoops.tsx` rendering live elapsed/countdown chips. A separate workspace-level scheduler (`lib/server/scheduler.ts` + `lib/shared/cron.ts` using `cron-parser` + `cronstrue`) drives durable cron *jobs* outside the agent runtime — that's the substrate `/schedule` reaches for when a loop outlives the session. The slash-command surface (`/loop`) is registered in `lib/shared/slash-commands.ts`.

## Decision
ALREADY_EXISTS. The CronCreate / CronList / CronDelete trio called out in `tools/ScheduleCronTool/prompt.ts` is faithfully reflected in Claudius — server-side handlers in `lib/server/session.ts`, client reducer in `lib/client/use-session.ts`, shared wire shape in `lib/shared/session-loops.ts`, browser views in `components/panels/widgets/ScheduledLoops.tsx` and `app/[workspaceId]/schedule/page.tsx`, and a durable cron substrate in `lib/server/scheduler.ts` + `lib/shared/cron.ts`. The `durable` flag and recurring-vs-one-shot split both round-trip; jitter and max-age are properties of the in-SDK Kairos tool itself and don't need extra surfacing on the Claudius side.
