# MCP / agent / deferred-tool delta reminders mid-session

**Source:** Claude Code TUI — observed in session
**Status:** ALREADY_EXISTS

## What it is
When MCP servers connect or disconnect, deferred-tool availability shifts, or agent types become available/unavailable, the harness prepends a targeted `<system-reminder>` to the next turn so the agent self-heals its mental model of which tools exist — e.g. `The following deferred tools are now available via ToolSearch ... with query "select:<name>[,<name>...]" to load tool schemas before calling them`, or `deferred tools are no longer available (MCP server disconnected): ...`. The reminder also carries the load-bearing guidance `Wait for connecting servers and search their tools once available. Do not report a capability as unavailable without first searching.` — which turns the post-reconnect pending-status race into correct behavior.

## Claudius today
`lib/server/session.ts` exports `mcpDeltaReminderBody(delta)` and a `McpDelta` type (`added` / `removed` / `disabled` / `reconnecting`) that emits the canonical clauses: `The following MCP servers are now available: ...`, `... were just reconnected and may still be connecting: ...`, `... are no longer available (disabled by the user): ...`, `... (removed from this session): ...`, plus the verbatim wait-and-search guidance. It is wired into `Session.reconnectMcp` / `toggleMcp` / `setMcpServers` (the `setMcpServers` path filters out `errors`-listed names so we don't tell the model "now available" for a failed handshake), queued through `lib/server/system-reminders.ts` under reminder kind `"mcp-delta"`, drained alongside `takeGoalReminder()` at the input-queue site, and pinned by `tests/unit/mcp-delta-reminder.test.ts`. Claudius names servers directly rather than mirroring the CLI's `select:<name>` deferred-tool / ToolSearch indirection (which Claudius doesn't have). Out of scope by design: spontaneous disconnects (server crash) and agent-type availability shifts — neither has an event signal in the SDK we wrap today, and the comment block at `lib/server/session.ts:404` calls this gap out explicitly.

## Decision
ALREADY_EXISTS. The parity surface is implemented: `mcpDeltaReminderBody` in `lib/server/session.ts` plus the `"mcp-delta"` queue entry in `lib/server/system-reminders.ts`, exercised by `tests/unit/mcp-delta-reminder.test.ts`. The only gap relative to the TUI is the deferred-tool / ToolSearch wording and spontaneous-disconnect / agent-type signals — both intentional, both documented inline; nothing to build unless the SDK starts surfacing those events.
