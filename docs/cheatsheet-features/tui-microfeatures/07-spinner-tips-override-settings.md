# spinnerTipsEnabled / spinnerTipsOverride config (+ per-tip cooldownSessions)

**Source:** Claude Code TUI — tip rotation
**Status:** ALREADY_EXISTS

## What it is
Two `settings.json` knobs control the CLI's spinner-tip rotation: `spinnerTipsEnabled` toggles the whole feature on/off, and `spinnerTipsOverride` takes `{ "excludeDefault": true, "tips": ["Custom tip"] }` to swap the built-in rotation for a custom list. Each built-in tip also carries its own `cooldownSessions` throttle (literals seen include 1, 5, 10, 15, 25, 30) so individual tips don't reappear every session; the TUI maps override tips to `custom-tip-${K}` objects with `cooldownSessions: 0` so user-supplied tips are never throttled.

## Claudius today
Claudius honors both CLI keys with the same shape. `ClaudeSettings` in `lib/server/settings.ts` declares `spinnerTipsEnabled?: boolean` and `spinnerTipsOverride?: { excludeDefault?: boolean; tips?: string[] }`, and `lib/server/session.ts` caches them at `start()` into `spinnerTipsConfig` and threads them into the per-subscriber `tips` SSE event via `selectTips({ spinnerTipsEnabled, spinnerTipsOverride })`. `lib/shared/tips.ts` is the single source of truth: `selectTips()` returns `[]` when `spinnerTipsEnabled === false`, maps override entries to `custom-tip-${i}` `Tip` objects (trimming and dropping empties), and either appends them or — when `excludeDefault: true` — replaces the built-in catalog with them. The renderer is `components/chat/SpinnerTip.tsx`, and `lib/client/useTipDismissals.ts` applies a per-tip "show ~20% as often" weighting (`DISMISSED_TIP_SHOW_PROBABILITY`) when the user clicks the dismiss control. The per-tip `cooldownSessions` literal is not modelled on `Tip`; the comment in `tips.ts` calls out that the dismiss-weighting is Claudius's analog of `cooldownSessions` ("show less, not never").

## Decision
ALREADY_EXISTS. The CLI-parity settings keys (`spinnerTipsEnabled`, `spinnerTipsOverride` with `excludeDefault` and `tips`) are already honored end-to-end through `lib/server/settings.ts` → `Session.spinnerTipsConfig` → `selectTips()` → the `tips` SSE event → `SpinnerTip.tsx`, and the override-tips-never-throttled semantic falls out of the dismiss-weighting being the only throttle. The remaining gap is the per-tip `cooldownSessions` literal on built-in tips — intentionally not mirrored, since `DISMISSED_TIP_SHOW_PROBABILITY` covers the same "show less" intent without needing a session counter. If a stricter cooldown is ever wanted, the natural shape is a `cooldownSessions?: number` field on the `Tip` type wired through `useTipDismissals`.
