# In-Process Teammate Task Type

**Source:** Claude Code TUI — tasks-teammate
**Status:** MISSING

## What it is
The TUI's background-task registry carves out a dedicated `in_process_teammate` kind, distinct from `local_bash` / `local_monitor` / `local_agent`, for the team-mode multi-agent surface: each teammate runs in its own `AsyncLocalStorage`-isolated context, identifies itself as `agentName@teamName`, flips between an `idle` and `active` state, threads its plan-mode approvals through the host session, and carries its own permission mode that Shift+Tab cycles independently while you're viewing that teammate (`tasks/InProcessTeammateTask/types.ts`: `// Permission mode for this teammate (cycled independently via Shift+Tab when viewing)\n  permissionMode: PermissionMode`). The string appears 59 times across the binary — concrete evidence that team mode is a real, shipped surface in the TUI build.

## Claudius today
Not surfaced in Claudius. The only acknowledgement is the reserved `TeammateIdle` hook in `lib/shared/hook-events.ts:78` ("When a multi-agent teammate goes idle."), which is registered in the hook event catalog but has no producer wired anywhere in `lib/server/session.ts` or the rest of the server. The `task_type` column in `lib/server/db-migrations/007_session_tasks.sql:35` and the `taskType` field in `lib/server/session-tasks-db.ts:57` / `lib/client/use-session.ts:2948` only carry the subagent-spawn flavor strings (`general-purpose`, workflow names, etc.) — no `in_process_teammate` kind appears anywhere, and `BackgroundTasksPanel.tsx` partitions strictly into `local_bash` / `local_monitor` / subagents (see `docs/cheatsheet-features/tui-microfeatures/84-background-task-pill-typed-labels.md`). The session-level `permissionMode` cycled by Shift+Tab in `components/chat/ModeSelector.tsx` is a single global mode for the host session, not a per-teammate one. There is no `agentName@teamName` identity scheme, no `AsyncLocalStorage` isolation (`grep` returns zero hits), and no team/coordinator gate distinct from the regular subagent flow (see `docs/cheatsheet-features/tui-microfeatures/32-stale-task-tools-teammate-nudge.md`).

## Decision
MISSING. The reserved `TeammateIdle` hook in `lib/shared/hook-events.ts:78` and the explicit "team-mode counterpart" call-out in `docs/cheatsheet-features/tui-microfeatures/README.md:26` together confirm that Claudius authors knew team mode existed but punted on hosting it. The `tasks/InProcessTeammateTask/types.ts` leak — with its `permissionMode: PermissionMode` field and 59-occurrence binary footprint — is concrete evidence that the TUI ships a real in-process teammate surface with its own permission cycle, plan-mode approval flow, and idle/active state. There is no host in Claudius today: the SDK does not expose teammate-spawn APIs to the agent SDK consumer, and adding a parallel teammate registry, a `agentName@teamName` identity, per-teammate `permissionMode` storage cycled by a teammate-scoped Shift+Tab, and an `in_process_teammate` kind in `BackgroundTasksPanel.tsx` would be a large net-new surface — worth tracking as a single epic gated on the SDK growing a stable team-mode interface, rather than scaffolding any individual piece (84's `N teams` pill label, 32's `task_reminder`, the `TeammateIdle` producer) in isolation.
