# User & Project Output-Styles Directories

**Source:** Claude Code TUI — plugins-output-styles
**Status:** PARTIAL

## What it is
The TUI auto-loads markdown files from `~/.claude/output-styles/*.md` (user scope) and `<projectCwd>/.claude/output-styles/*.md` (project scope) as selectable output styles, with project entries shadowing user entries of the same name. The leak comment in `outputStyles/loadOutputStylesDir.ts` spells out the contract: "Loads markdown files from .claude/output-styles directories throughout the project and from ~/.claude/output-styles directory and converts them to output styles. Each filename becomes a style name, and the file content becomes the style prompt. The frontmatter provides name and description." Same shape as the existing Skills/Agents directory loaders — filename → id, body → prompt, frontmatter → `name` + `description`.

## Claudius today
Claudius models `outputStyle` as a single settings string. `lib/server/settings.ts` line 19 declares `outputStyle?: string` on `ClaudeSettings`, and the settings UI exposes a fixed enum: `app/settings/page.tsx` line 31 defines `const OUTPUT_STYLES = ["default", "explanatory", "concise", "developer"]`, rendered by the "Output style" `<select>` at lines 449–463 and persisted on the `outputStyle` key (line 726). There is no scanner for `.claude/output-styles/*.md` — compare `lib/server/skills.ts` (which resolves `<cwd>/.claude/skills/<name>/SKILL.md` and `~/.claude/skills/<name>/SKILL.md`, lines 16–17) and the Agents/Skills pages (`app/[workspaceId]/skills/page.tsx` lines 13–14, `app/[workspaceId]/agents/page.tsx` lines 13–14) that already ship the user/project scope-toggle pattern this feature would reuse. The `outputStyle` field in `lib/server/settings.ts` is plumbed straight through to the SDK settings file, so any styles the SDK itself discovers on disk would already be honored at runtime — but Claudius does not surface them in the picker, nor edit them, and the dropdown options are hardcoded to the four built-ins.

## Decision
PARTIAL. The settings slot already exists end-to-end (`lib/server/settings.ts` line 19, `app/settings/page.tsx` lines 31 + 449–463), so the SDK will honor any user/project markdown styles at runtime. To match the leak (`outputStyles/loadOutputStylesDir.ts`), Claudius would need a `lib/server/output-styles.ts` scanner mirroring `lib/server/skills.ts` (resolve `~/.claude/output-styles/*.md` and `<projectCwd>/.claude/output-styles/*.md`, parse frontmatter for `name`/`description`, project shadows user by filename id), an `app/api/output-styles` list/read/write/delete route family, and either: (a) replace the hardcoded `OUTPUT_STYLES` enum in `app/settings/page.tsx` with the live list, or (b) ship a dedicated `app/[workspaceId]/output-styles/page.tsx` editor reusing the Skills/Agents `ScopeToggle` shape so users can author styles in-app instead of editing the markdown files by hand.
