# 'ultraplan' prose keyword auto-launches cloud planning session

**Source:** Claude Code TUI ŌĆö input keyword nudge
**Status:** PARTIAL

## What it is
Typing `ultraplan` anywhere in a prompt (not just as `/ultraplan <prompt>`) trips a TUI banner announcing that the turn will run as a remote ultraplan session in Claude Code on the web, and emits a `tengu_ultraplan_keyword` telemetry event. The binary strings spell out the contract: `Usage: /ultraplan \<prompt\>, or include "ultraplan" anywhere ... ultraplan-active ... This prompt will launch an ultraplan session in Claude Code on the web ... tengu_ultraplan_keyword`.

## Claudius today
The slash command is registered ŌĆö `lib/shared/slash-commands.ts` line 174 has `{ id: "ultraplan", name: "ultraplan", description: "Browser-based plan, then execute.", category: "skill", handler: "sdk", argsHint: "<prompt>" }`. The prose-keyword detector also ships: `components/chat/PromptInput.tsx` line 140 adds an `ultraplan` entry to `KEYWORD_HINTS` with `pattern: /(?<![/\w])ultraplan\b/i` (anchored to the bare word, not `/ultraplan` and not `ultraplanning`) and the composer-footer label `ultraplan-active ŌĆö run with /ultraplan to launch a browser planning session`. `lib/server/system-reminders.ts` line 31 even reserves an `"ultraplan-prose"` kind in `ReminderKind`. What is *not* wired is the second half of the contract: there is no `queueReminder(this, "ultraplan-prose", ...)` call anywhere in `lib/server/session.ts` (only the sibling `ultrathink-prose` is actually enqueued at line 2049), no banner copy promising a remote launch in Claude Code on the web, no actual remote/cloud planning host the local session can hand off to, and no `tengu_ultraplan_keyword` (or Claudius-side) telemetry event.

## Decision
PARTIAL. The detector and a composer hint are in place ŌĆö `components/chat/PromptInput.tsx` line 140 already surfaces `ultraplan-active` when the user types the bare word, and `/ultraplan` runs through the SDK skill handler. The remaining gap is the TUI's stronger framing: the "This prompt will launch an ultraplan session in Claude Code on the web" banner (and the underlying remote-handoff), plus wiring the reserved `ultraplan-prose` reminder kind in `lib/server/system-reminders.ts` through a `queueReminder` call alongside the existing `ultrathink-prose` one in `lib/server/session.ts`, and emitting a matching telemetry event. Worth adding in the same composer banner row in `components/chat/PromptInput.tsx` (or as a sibling to `PlanModeBanner.tsx`) once a "Claude Code on the web" target exists for Claudius to point at.
