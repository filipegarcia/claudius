# Background-Task Pill with Type-Specific Labels

**Source:** Claude Code TUI — tasks-teammate
**Status:** PARTIAL

## What it is
The TUI footer's background-task pill collapses to a type-specific compact label whenever every running task shares the same kind, instead of always printing the generic `N background tasks`. The lookup in `tasks/pillLabel.ts` resolves to `1 shell` / `N shells` for `local_bash`, `1 monitor` / `N monitors` for `local_monitor`, `1 local agent` / `N local agents` for `local_agent`, `1 team` / `N teams` for in-process teammates, `1 background workflow`, a diamond-prefixed `1 cloud session` / `N cloud sessions` (`return n === 1 ? \`${DIAMOND_OPEN} 1 cloud session\` : \`${DIAMOND_OPEN} ${n} cloud sessions\``), and a single `dreaming` for an idle agent that's still thinking. Mixed kinds fall back to `N background tasks`.

## Claudius today
The right-rail header in `components/panels/BackgroundTasksPanel.tsx` already partitions running work by SDK kind — `PROCESS_TASK_TYPES = new Set(["local_bash", "local_monitor"])` (line 101), `iconForType` maps `local_bash → Terminal` and `local_monitor → Activity` (lines 123-124), and `local_agent` / `local_workflow` stay in the `subagents` bucket per the comment on line 99. The header itself prints only the generic count: ``aria-label={`${runningBg} background ${runningBg === 1 ? "task" : "tasks"} running — show them`}`` (line 364) with the visible text `{runningBg} running` (line 368) and a sibling `{attention}` counter (line 384). There's no collapse-to-shared-kind: even when every running entry is a shell or every entry is a workflow, the rail still says "N running" with the Bot/Terminal icon switch as the only kind hint. `lib/client/task-status.ts` and `collectStoppableTaskIds` carry the disjoint subagents / process-task / shell partitions that a typed label would key off. Claudius has no "cloud session" or in-process "team" concept — there's no remote-execution surface (`grep` returns zero hits for cloud-session) and no AM()/team-mode gate (see `docs/cheatsheet-features/tui-microfeatures/32-stale-task-tools-teammate-nudge.md`), so the diamond and `N teams` variants have no host.

## Decision
PARTIAL. The kind-aware bookkeeping is in place — `PROCESS_TASK_TYPES`, the per-kind icon switch, and the subagents/processes/shells partitions in `collectStoppableTaskIds` — but the type-specific compact label leaked in `tasks/pillLabel.ts` is not surfaced. A targeted follow-up is a small pure helper alongside `lib/client/task-status.ts` that takes `(subagents, runningProcessTasks, runningBashes)` and, when all entries collapse to one kind, returns one of `1 shell`/`N shells`, `1 monitor`/`N monitors`, `1 local agent`/`N local agents`, or `1 background workflow` (workflows can be filtered out of `subagents` by `block.name === "Workflow"`, which `isBackgroundedToolUse` already special-cases at line 64 of `task-status.ts`); wire it into the header at lines 364/368 in `components/panels/BackgroundTasksPanel.tsx`, falling back to today's `N running` for mixed kinds. The `dreaming` state could ride the existing `pending && runningBg === 0` branch. The `team` and diamond `cloud session` variants stay out of scope until Claudius grows a teammates or cloud-execution surface.
