## Summary

CC 2.1.195 is a quality-of-life / bug-fix release. The substantive items are: a hook-matcher fix for hyphenated tool names, voice dictation improvements (macOS silence capture, CJK auto-submit, Linux microphone detection), background-agent reliability fixes, plugin install-consent tightening, and a remote-session provisioning checklist. None of these have a Claudius browser-UI analog that needs reimplementing. Claudius has its own voice subsystem but intentionally has no auto-submit (the composer holds that decision), so the CJK voice fix does not apply. The plugins enable/disable fix is in the CC CLI's interactive command path, not in Claudius's own settings-write path. Hook-matcher evaluation happens inside the SDK; Claudius's hooks.ts is pure CRUD. The headline risk for review: the hook-matcher fix (Bucket A) assumes SDK 0.3.195 carries the same fix — if it does not, users with hyphenated matcher identifiers (`code-reviewer`, `mcp__brave-search`) will still see incorrect substring-matching through Claudius.

## Changelog classification

- `[A — via SDK updater] Fixed hook matchers with hyphenated identifiers accidentally substring-matching` — hook execution and matcher evaluation live entirely inside the SDK (Claudius's `hooks.ts` is read/write-only CRUD; `session.ts` passes hooks to the SDK's `query()` and never evaluates matchers itself). Note: the SDK 0.3.195 run-notes do not explicitly confirm this fix; see Risks.
- `[A — via SDK updater] Fixed /plugin Enable/Disable not working when plugin.json name differs from marketplace entry name` — Claudius's own toggle UI calls `setEnabled(pluginId)` using the SDK-reported `installedPlugin.name` from `reloadPlugins()`, which reads `plugin.json`'s `name` field directly; it does not go through the CLI's `/plugin` command path that had the bug. The native-handler `/plugin` command (for installs) flows through the SDK, which carries the fix.
- `[A — via SDK updater] Fixed background jobs disappearing from claude agents or losing data when written by a newer Claude Code version` — this is about the SDK/CLI internal agent-job file format; the SDK 0.3.195 update carries any format-compatibility fix.
- `[C — CLI/terminal only] Added CLAUDE_CODE_DISABLE_MOUSE_CLICKS to disable mouse click/drag/hover in fullscreen mode while keeping wheel scroll` — terminal full-screen mouse-event control; Claudius is a browser app with no terminal mouse surface.
- `[C — CLI/terminal only] Fixed voice dictation on macOS capturing silence in long-running sessions after the default input device changes` — macOS-specific audio-device-change detection in the CC CLI's SoX-backed capture pipeline; Claudius's voice uses the browser's `getUserMedia` API, which does not have this input-device-change behavior.
- `[C — CLI/terminal only] Fixed voice dictation auto-submit never firing for languages written without spaces (Japanese, Chinese, Thai)` — CC CLI's auto-submit heuristic (keys on whitespace) does not exist in Claudius; `useVoice.ts` delivers `TranscriptText` / `TranscriptInterim` events to the composer (`PromptInput.tsx`), which never auto-submits — the user must press Enter.
- `[C — CLI/terminal only] Fixed external plugins enabled only by project .claude/settings.json not requiring explicit install consent on every loader path` — CC CLI's interactive marketplace-install consent flow; Claudius's plugins page reads `enabledPlugins` from settings files and delegates install to the SDK via `/plugin install`, with no independent consent gate to fix.
- `[C — CLI/terminal only] Fixed reopening a crashed background task showing a blank screen for up to 5 seconds` — CC CLI TUI background-task reconnect screen; no equivalent in Claudius's browser session surface.
- `[C — CLI/terminal only] Fixed background agent daemons running unreachable when the control socket fails to start` — CC CLI background-agent Unix-socket daemon; Claudius runs sessions in-process inside the Next.js server with no OS-level daemon or control socket.
- `[C — CLI/terminal only] Improved voice mode on Linux: distinguishes no microphone from SoX not installed` — Linux SoX audio-capture path in the CC CLI; no Claudius analog.
- `[C — CLI/terminal only] Improved claude agents completed list to fill available vertical space; header compacts on short terminals` — CC CLI TUI layout for the `claude agents` screen; Claudius's agents page is a browser UI with no terminal layout constraints.
- `[C — CLI/terminal only] Improved Remote session startup with a provisioning checklist while the container starts` — CC CLI's remote-container session startup; Claudius has no remote container session concept.

## Risks / follow-ups

- **Hook-matcher fix may not be in SDK 0.3.195.** The SDK 0.3.195 run-notes mention only `reinitialize()` and `commands_changed` — they do not confirm the hyphenated-identifier hook-matcher fix. If the fix is only in the CC CLI binary and not in the SDK's hook-execution path, Claudius users who have hook matchers with hyphenated identifiers (e.g. `matcher: "code-reviewer"`) will still see incorrect substring-matching through the SDK. To verify: write a hook with `matcher: "code-review"` and test whether it incorrectly fires for `code-reviewer`. If confirmed, a follow-up bucket-B item would be needed: implement a Claudius-side matcher shim or surface a UI warning about hyphenated matchers.
- **Voice CJK auto-submit near-miss.** The CJK voice fix was initially a candidate for bucket B because Claudius has its own voice system. It was classified C because Claudius deliberately has no auto-submit: the composer holds the submit decision and the user must press Enter. If a future feature request adds auto-submit to Claudius's voice, the CJK (space-less utterance boundary) behavior would need to be ported at that time.
- **Plugin name-mismatch data migration.** Users who installed plugins via the old CC CLI (pre-2.1.195) and whose `plugin.json` name differs from the marketplace entry name may have stale keys in `enabledPlugins` (e.g. `marketplace_name: true` instead of `plugin_json_name: true`). Claudius's toggle UI uses the SDK-reported name, so toggling such a plugin OFF would attempt to delete the wrong key and appear to not work. This is a legacy-data problem, not a code bug, and is resolved for new installs by CC 2.1.195. A one-time migration scan (`enabledPlugins` keys vs SDK-reported installed names) could close the gap for existing users — deferred.
