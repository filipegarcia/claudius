## Summary

Claude Code 2.1.217 is mostly bug fixes plus two engine-side subagent
defaults (concurrency cap, no-nested-spawn-by-default) that land in
Claudius via the sibling SDK-update pipeline's `@anthropic-ai/claude-agent-sdk`
0.3.217 bump — which rides in this same PR (#161) as its own commit
(`chore(deps): bump claude-agent-sdk to 0.3.217`); the parity commits
below add no dependency bump of their own, same combined-PR shape as the
2.1.216 release (#160). Two items had a real Claudius-side
product surface and were reimplemented this release: emoji shortcode
autocomplete in the prompt composer (`:heart:` → ❤️, `:hea` → suggestions,
gated by the new `emojiCompletionEnabled` setting), and a small update to
the token-expiry-warning threshold now that upstream has revealed its own
value (5→3 days). The transcript-write-failure / session-saving-off
warning entry looked like bucket B at first glance but turned out to be
bucket A on closer reading of the SDK: it rides the SDK's existing generic
`system/informational` message channel (level: "warning"), which Claudius
has rendered end-to-end since 0.3.178 — no code was needed. The headline
risk to flag for review: while building the emoji picker, I found and
fixed a real pre-existing bug in `AtMentionPicker`/`SlashCommandPicker`
where Enter/Tab-to-select could double-fire into the composer's own
submit/Tab-indent handling (see Risks below) — worth a second look since
it touches already-shipped composer behavior, not just new code.

## Implemented (bucket B)

- `[extend: chat composer] Emoji shortcode autocomplete` — typing `:heart:` swaps it inline for ❤️; typing a partial (`:hea`) opens a suggestion dropdown (same shape/keyboard contract as the existing slash-command and @-mention pickers), gated by a new `emojiCompletionEnabled` user setting (default on). Files touched: `lib/shared/emoji-shortcodes.ts` (new — pure trigger-detection/filter/lookup logic + curated shortcode table), `components/chat/EmojiShortcodePicker.tsx` (new — the dropdown, modeled on `AtMentionPicker.tsx`), `components/chat/PromptInput.tsx` (trigger detection in `refreshPickerState`, auto-replace-on-closing-colon in the textarea `onChange`, `insertEmojiShortcode`, picker mount + mutual-exclusion with the slash/@-mention pickers), `lib/client/useEmojiCompletionEnabled.ts` (new hook, modeled on `useDisableAutoMode.ts`), `components/chat/ChatSurface.tsx` (wires the hook into the main composer), `lib/server/settings.ts` (`emojiCompletionEnabled?: boolean` schema entry), `app/settings/page.tsx` (new `ToggleRow` in the Chat section + `KNOWN_KEYS`/search-keyword updates). Rejected alternative: a brand-new "Emoji" settings page/dedicated panel — overkill for a single on/off toggle that's a variation on the existing Chat section's other composer toggles (`promptSuggestionEnabled`, `sessionRecapEnabled`).
- `[extend: token-expiry warning threshold] Login-expiry warning window 24h → 3 days` — `lib/server/token-expiry.ts`'s `TOKEN_EXPIRY_WARNING_WINDOW_MS` updated from a documented 24h guess to the now-known upstream value (3 days), with the doc comment updated to cite CC 2.1.217 as the source. No UI changes (the existing `TokenExpiringPanel` UI is unchanged, only *when* it fires shifts) — see Risks for the "why not touch the UI" call.

Incidental fix (not a changelog item, found while building the above): `AtMentionPicker.tsx` and `SlashCommandPicker.tsx`'s keyboard handlers were missing `e.stopPropagation()` on Tab/Enter/Escape/Arrow selection. Without it, the *same* keydown that a picker uses to select an item could still reach `PromptInput`'s own `onKeyDown` (bubble phase) after the picker's `onSelect` had already reset its query state — misfiring the Enter-submits or Tab-indents fallback on the very keystroke meant to just insert the mention/command. Confirmed via a throwaway repro against the existing (untouched) `AtMentionPicker`: pressing Enter to select a fully-loaded `@package` match submitted/cleared the composer instead of inserting the mention. Fixed in all three pickers (including the new `EmojiShortcodePicker`) by adding `stopPropagation()` alongside the existing `preventDefault()`.

## Risks / follow-ups

- **Emoji picker delivery shape** — chose `[extend: chat composer]` (a third picker alongside slash/@-mention) over a `[new screen]` approach (e.g. a dedicated emoji-browser panel). Rejected the new-screen shape because the feature is explicitly a composer-inline autocomplete in the upstream changelog, not a browsing UI — matching the existing picker pattern keeps the affordance consistent with how Claudius already teaches this interaction (Tab to insert, arrow to navigate).
- **Login-expiry threshold — conservative vs. maximal interpretation** — conservative (implemented): update the internal constant only, leave `TokenExpiringPanel`'s UI and the setting surface untouched, since this is a heuristic threshold rather than a user-configurable value upstream. Maximal alternative (not implemented): expose the warning window as its own Claudius setting (e.g. `tokenExpiryWarningDays`) so users could tune it — rejected as speculative scope the changelog entry doesn't ask for; upstream doesn't expose this as a setting either.
- **`system/informational` severity styling** — confirmed Claudius already renders the SDK's transcript-write-failure / session-saving-off warnings (bucket A), but the client (`use-session.ts` ~line 3783) renders every `informational` message as the same neutral pill regardless of `level` ("info" vs "warning" vs "suggestion" render identically today). Not fixed here since it's a pre-existing, general gap unrelated to this specific changelog entry (it predates 2.1.217 and affects every `informational` message, not just this one) — flagged for a follow-up pass to differentiate `level` visually (e.g. amber tint for "warning").
- **Ambiguous/skipped item** — "Capped the frontend-design plugin suggestion tip at 3 lifetime impressions" marked skip: no Claudius analogue exists (see classification above). Disambiguated via direct code audit (`lib/server/plugins.ts`, `app/plugins/page.tsx`, grep for suggestion/tip/impression-cap patterns), not `gh api`/docs — the upstream entry is self-explanatory enough that the ambiguity is purely "does Claudius have this system at all," which the audit answered (no).
- **Bug fix spot-check, not parity** — "Fixed background session isolation not canonicalizing symlinked working directories" (elided as a pure bug fix) is worth a quick independent look at Claudius's own workspace/session path handling (see `CLAUDE.md`'s path-safety conventions) even though it isn't a parity item — flagging per the classification note.
- **Pre-existing e2e parallel-load flakes (not introduced here)** — a full `test:e2e` run flaked four specs unrelated to this release: `notifications-multi-tab.spec.ts`, `session-tabs-pin.spec.ts`, `tool-call-answer-pill.spec.ts`, and `workspace-route-memory.spec.ts`. All four pass deterministically when re-run in isolation (`23/23`), which both confirms they're load-induced flakes and clears this release's one cross-cutting change (the token-expiry `24h → 3d` window, which widens when `TokenExpiringPanel` renders) as a cause — the route/click-sensitive specs pass with full chrome and no panel-induced layout shift. Same class of flakiness documented in the 2.1.216 run-notes; the shared per-cwd tab store (which the emoji spec already stubs via `open-tabs`, and which `session-tabs-pin` reads) is a known cross-worker contention point. No fix attempted here — flagged for the systemic-e2e-isolation follow-up rather than this parity PR.
- **Incidental picker bug fix** — the `stopPropagation()` fix to `AtMentionPicker.tsx`/`SlashCommandPicker.tsx` (see Implemented) touches already-shipped composer behavior, not just this release's new code. It's a straightforward, narrowly-scoped fix (two lines per key branch), and now has its own dedicated regression spec (`tests/e2e/composer-picker-key-select.spec.ts`) independent of the emoji feature — but it's still worth a specifically careful review since it changes event-propagation behavior in a widely-used, heavily-commented component, and was never caught by the existing test suite before this pass.
