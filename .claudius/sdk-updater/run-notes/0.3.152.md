## Summary

The SDK jumped from 0.2.132 to 0.3.152, a large minor version that replaces the `TodoWrite` tool with a structured **Task tools** API (`TaskCreate`, `TaskUpdate`, `TaskList`), adds a new `MessageDisplay` hook event, and ships several type-only enrichments (new `SDKAssistantMessage` fields, `model_not_found` error variant, `SessionStartHookSpecificOutput` additions, `api_error_status` on result payloads). The removed `unstable_v2_createSession` surface was not used in Claudius. Headline risk: the Task tools change requires a two-sided migration — client (`use-session.ts`) and server (`session.ts` snapshot) — plus backward-compat retention of the `TodoWrite` path for sessions created before this update.

## SDK changelog highlights

- **Task tools (TaskCreate / TaskUpdate / TaskList) replace TodoWrite** [shipped] — `TodoWriteInput` is still present in types but deprecated; new `TaskCreateInput`, `TaskUpdateInput`, `TaskListInput` / `TaskListOutput` types added to `sdk-tools.d.ts`. Claude now calls `TaskCreate` when it wants to add a to-do item, and `TaskUpdate` to mark it done or delete it.
- **`MessageDisplay` hook event** [shipped] — `"MessageDisplay"` added to SDK's `HOOK_EVENTS` array and `HookEvent` union; `MessageDisplayHookInput` / `MessageDisplayHookSpecificOutput` types exported. Claudius's `lib/shared/hook-events.ts` mirror must be updated.
- **MCP non-blocking connect by default** [type-only] — MCP servers now start in a `'pending'` state until connection resolves; `useMcp.ts` already had `'pending'` in the status union, no runtime change needed.
- **`SDKAssistantMessageError` gains `'model_not_found'`** [type-only] — new variant alongside `'invalid_request'`; no Claudius error-handling code switches on this value.
- **`SDKAssistantMessage` new fields** (`request_id?`, `subagent_type?`, `task_description?`) [type-only] — informational metadata; not consumed by Claudius today.
- **`SessionStartHookSpecificOutput` new fields** (`sessionTitle?`, `reloadSkills?`) [type-only] — hooks are CLI-driven in Claudius (settings.json), no in-process handler to update.
- **`SDKResultSuccess.api_error_status?: number | null`** [type-only] — surface for HTTP status on API errors in result payloads; not consumed by Claudius.
- **`unstable_v2_createSession` / `SDKSession` API removed** [skipped — not used in Claudius codebase; zero references confirmed].

## Code changes

- **`lib/shared/hook-events.ts`** — Added `"MessageDisplay"` to `HOOK_EVENT_NAMES` const array and added a matching `HookEventSpec` entry to `HOOK_EVENTS`, keeping Claudius's mirror in sync with the upstream SDK hook enum.
- **`lib/client/use-session.ts`** — Added `TaskCreate` / `TaskUpdate` / `TaskList` handlers alongside the existing `TodoWrite` handler in the `tool_use` / `tool_result` processing path. `TaskCreate` pending blocks are keyed by `tool_use_id` in a local map; on the matching `tool_result` the `task.id` from the response payload is used as the stable key and the item is promoted into the `latestTodos` state array. `TaskUpdate` looks up by `taskId` from the tool input and applies status / subject changes in-place. `TodoWrite` handler retained for backward compatibility with older sessions. `perTool` picker map extended with `TaskCreate: ["subject"]`, `TaskUpdate: ["taskId"]`, `TaskList: []`.
- **`lib/server/session.ts`** — Extended `captureSnapshotState` to accumulate Task tool calls using the same pending-map pattern as `trackScheduledLoops`: assistant `tool_use` blocks for `TaskCreate` are stored in `pendingTaskCreates` keyed by `tool_use_id`; matching `tool_result` blocks promote them into `latestTodosSnapshot` using the result's `task.id`. `TaskUpdate` events mutate the snapshot in-place by `taskId`. `TodoWrite` capture path retained unchanged.

## Risks / follow-ups

- **Backward-compat window for TodoWrite**: sessions started before this deploy will replay `TodoWrite` tool calls on reconnect; the retained handler covers this. Once all active sessions have rolled over the `TodoWrite` path can be removed (follow-up ticket).
- **TaskCreate result parsing**: the server-side accumulation in `captureSnapshotState` depends on the result payload being `{ task: { id: string } }` — if the SDK changes this shape in a patch the snapshot will silently fail to promote. Add a unit test in a follow-up.
- **`model_not_found` error display**: Claudius renders `SDKAssistantMessageError` inline but does not branch on the variant. A `model_not_found` error will show as a generic error bubble. Low-priority UX follow-up.
- **`sessionTitle` / `reloadSkills` in SessionStart hooks**: if users configure a SessionStart hook that returns `sessionTitle`, Claudius will silently ignore it. Follow-up to wire `sessionTitle` into the session rename flow.
