## Summary

`@anthropic-ai/claude-agent-sdk@0.3.208` is a small bug-fix-only release:
seven fixes, all internal to the SDK's process/hook/stdin plumbing, none
of which add, remove, or reshape a public export. Audited every item
against Claudius's actual call sites (`lib/server/session.ts` is where
all of them would land, since that's the only place Claudius touches
hooks, `extraArgs`-shaped options, `createSdkMcpServer`, or process
spawning). Result: **zero Claudius code changes required.** Claudius
registers its own `PreToolUse`/`PostToolUse`/`CwdChanged` hooks (not
`UserPromptSubmit`), never passes `extraArgs`, never resumes with a
dash-leading value (`resume` is always Claudius's own session UUID),
and its one `createSdkMcpServer` use (`buildGoalMcpServer`) is a
synchronous in-process tool untouched by either the doc fix or any
timeout knob. Every fix in this release is a correctness/robustness fix
in code paths Claudius calls but doesn't reimplement, so the corrected
behavior is inherited automatically with no adaptation needed. No
headline risk — this is a clean, no-op bump for us.

## SDK changelog highlights

- `[skipped — SDK-internal, no call-site]` Fixed a caller abort during a
  pending SDK hook callback being converted into hook success, which let
  `PreToolUse`-gated tools execute after the abort. Claudius *does*
  register a `PreToolUse` hook (`lib/server/session.ts` ~2242, used for
  worktree-cwd detection and mid-turn reminder draining), so this is the
  one item worth checking closely — but the bug and fix are entirely in
  the SDK's own hook-dispatch race handling (how it converts a pending
  callback to a result when `abortController.abort()` fires
  concurrently), not in the hook callback shape or return contract
  Claudius implements. Our hook still returns `{ continue: true }` /
  `hookSpecificOutput` exactly as before; we get the corrected
  abort-race behavior for free with no code change.
- `[skipped — SDK-internal, no call-site]` Fixed a per-query resource
  leak in the SDK's process tracking when spawning the CLI fails
  (nonexistent or inaccessible executable path). Internal to the SDK's
  process-tracking bookkeeping; Claudius spawns via the SDK's
  `query()`/`Query` surface and never touches process-tracking
  internals directly. Grepped for `pathToClaudeCodeExecutable` /
  process-tracking hooks in `session.ts` — none.
- `[skipped — SDK-internal, no call-site]` Fixed an SDK
  `UserPromptSubmit` hook callback exceeding its timeout killing the
  entire query with an empty error; it now blocks the prompt with a
  clear timeout message and the session continues. Grepped the whole
  repo (`rg -n "UserPromptSubmit"`) — Claudius never registers a
  `UserPromptSubmit` hook. The only repo references are the hook-event
  catalog (`lib/shared/hook-events.ts`, describing what a *user's own*
  `settings.json` hook could do) and a UI comment in
  `use-session.ts`/an exploration doc — neither is a Claudius-owned
  hook implementation. If a user's own `settings.json` defines a
  `UserPromptSubmit` hook, they now transparently get the improved
  timeout behavior with no Claudius-side change needed.
- `[skipped — no call-site]` Fixed `extraArgs` values that look like
  flags (e.g. `resume: '--version'`) being parsed as their own CLI
  flags; dash-leading values are now bound with equals-form argv.
  Grepped for `extraArgs` across `lib/` and `app/` — zero hits, Claudius
  never sets `options.extraArgs`. Claudius does pass its own `resume`
  option (`session.ts:2349`, `resume: this.resumeFrom`), but
  `resumeFrom` is always Claudius's own session UUID
  (`readonly resumeFrom?: string`, threaded from `opts.resume` at
  session construction) — never user-typed text, so it can never
  collide with a CLI flag shape either way.
- `[skipped — SDK-internal, no call-site]` Fixed an abort-listener leak:
  streaming queries sharing one `AbortController` no longer accumulate
  `abort` listeners on its signal after each completed query. Claudius
  creates a fresh `AbortController` per session
  (`private abortController = new AbortController()`) and a fresh one
  per recap turn (`this.recapAbort = new AbortController()`), never
  sharing a single controller across multiple independent streaming
  queries the way this leak required — so this class of leak wasn't
  reachable from Claudius's usage pattern to begin with, and the fix
  needs no adaptation.
- `[skipped — doc-only, no call-site]` Fixed `createSdkMcpServer` docs
  pointing at a nonexistent env var; the MCP tool-call timeout knob is
  `MCP_TOOL_TIMEOUT`. Claudius's only `createSdkMcpServer` use
  (`buildGoalMcpServer`, `session.ts:3220`) is a synchronous in-process
  tool (`report_goal_achieved`) with no external I/O, so neither the old
  nor corrected env var name is ever relevant to it. Pure doc fix, no
  type or behavior change either.
- `[skipped — SDK-internal, no call-site]` Fixed an uncaught exception
  when writing to stdin after the Claude Code subprocess has exited.
  Internal to the SDK's own child-process stdin-write path; Claudius
  never writes to the CLI subprocess's stdin directly (all interaction
  is through the SDK's typed `Query` surface) — the `stdin` hits found
  in `session.ts` are all about a *different* subprocess entirely (the
  `sudo` password-prompting wrapper for the Bash tool, ~line 3103),
  unrelated to the SDK's own CLI child process.

**Model-identity audit (required before any item can be marked skipped

## Code changes

- No code changes required. All seven 0.3.208 fixes are corrections to
  SDK-internal plumbing (hook-dispatch abort races, process-tracking
  leaks, `extraArgs` argv parsing, abort-listener accumulation, a
  doc-only env-var rename, a stdin-write exception) that Claudius
  consumes transparently through the typed `Query`/`Options`/hooks
  surface. None of them change a public type shape, and none of them
  are reachable from how Claudius actually calls the SDK: Claudius
  never sets `extraArgs`, never shares one `AbortController` across
  streaming queries, never writes to the CLI's stdin directly, and its
  one `createSdkMcpServer` tool is synchronous/in-process. The one item
  with a real Claudius-owned call site — `PreToolUse` hooks — is fixed
  entirely on the SDK's internal abort-race handling, not the hook
  callback contract Claudius implements, so no adaptation is needed
  there either. See "SDK changelog highlights" above for the per-item
  grep-backed audit.
