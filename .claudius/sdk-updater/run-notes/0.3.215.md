## Summary

`@anthropic-ai/claude-agent-sdk@0.3.215`'s only changelog line is
"Updated to parity with Claude Code v2.1.215" — no itemized bullets.
I verified what that delta actually is by `npm pack`-ing both 0.3.214
and 0.3.215 and diffing the extracted tarballs byte-for-byte: all six
`.d.ts` files (`sdk.d.ts`, `bridge.d.ts`, `browser-sdk.d.ts`,
`sdk-tools.d.ts`, `agentSdkTypes.d.ts`, `extractFromBunfs.d.ts`) are
**identical** between the two versions — Claudius's entire public API
surface as a TypeScript consumer of this package is unchanged.
`browser-sdk.js` differs only in a version-string comment.
`sdk.mjs`/`bridge.mjs` differ by ~150 bytes total, and folding out the
version strings shows the remaining diff is purely minifier
parameter-name churn from a re-bundle (`eot`→`not`, etc.) across
vendored dependencies (mime-db, async, ajv) — no logic added or
removed. `manifest.json` shows the real "parity" delta lives in the
native `claude` binaries themselves (new checksums/sizes/build date
per platform) — a surface Claudius never touches directly, since it
always drives the CLI through the SDK's typed `query()`/`Query` API.
**Result: zero Claudius code changes required.** This was independently
re-verified by a skeptic sub-agent tasked with finding any reason a
change was needed, who confirmed the same byte-level findings and
additionally flagged that Claudius's model-identity surfaces
(`app/api/models/route.ts` etc.) are upgrade-safe by construction —
they source from `Query.supportedModels()` at runtime with a
self-documented bare-alias static fallback, so even a same-name model
change baked into the new binary wouldn't require a Claudius edit.
No headline risk — this is a clean, no-op bump for us.
