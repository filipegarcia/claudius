## Summary

SDK 0.3.186–0.3.187 brings three protocol additions: (1) `agent_id` on
`can_use_tool` control requests so background subagents forward permission
prompts to `canUseTool` instead of auto-denying; (2) `ReadMcpResourceDirTool`
as a new dedicated MCP tool type for directory listing; and (3)
`rewind_conversation` as a new control-request type for conversation-level
rewind with durable resume anchors. 0.3.187 adds `sandbox.credentials` to
the `SandboxSettings` type, allowing credential files and env-vars to be
denied inside sandboxed commands. Claudius ships one visible change: the
permission-prompt modal now shows a "Subagent" badge and the agent ID when a
permission request originates from a background subagent, which was previously
impossible because background agents auto-denied all tool calls. No breaking
changes; all four gates green.

## Code changes

- `lib/shared/events.ts` — Added `agentId?: string` to `PermissionRequestEvent`.
  Lets the browser know when a permission request originates from a
  background subagent.
- `lib/server/session.ts` — Forward `ctx.agentID` to the `PermissionRequestEvent`
  object in the `canUseTool` callback. One-liner addition.
- `components/chat/PermissionPrompt.tsx` — Render a small "Subagent" badge
  in the permission header when `request.agentId` is present, with
  `data-testid="permission-agent-badge"` for the Playwright spec.

## Risks / follow-ups

- `sandbox.credentials` is a new feature surface not yet exposed in
  Claudius's sandbox toggle UI. A follow-up task should add UI to configure
  which credential files/env-vars to deny in sandboxed commands, wired
  through `SessionOptions.sandbox.credentials`.
- The `rewind_conversation` control request has no `Query` host method yet.
  Watch for a future SDK release that adds `query.rewindConversation()` —
  Claudius's existing rewind route (`/api/sessions/[id]/rewind`) handles
  file rewind and would need a companion endpoint for conversation rewind.
