## Summary

0.3.160 delivers two SDK changes: a parity bump to Claude Code v2.1.159 (0.3.159) and a bug fix that stops hook callbacks from swallowing abort signals (0.3.160). The abort-signal fix is the substantive change: when `abortController.abort()` fires while a PostToolUse hook is running, the SDK now ends the turn cleanly with a final `result` message instead of hanging the calling process. Claudius already handles `result` messages correctly in its `consume()` loop, and its `PostToolUse` hook (which hashes files for the post-write snapshot tracker) has no workarounds that need unwinding. No Claudius code changes are required; all four gates (lint, vitest, build, Playwright) pass green on the bumped package alone.
