## Summary

This bump covers `main`'s prior merge base at 0.3.216 up to 0.3.217.
The only content genuinely new to *this* PR is the 0.3.217 changelog
(0.3.215 was a no-op parity bump and 0.3.216's one Claudius-relevant
item, `rewindFiles`'s `skippedLinks` count, already shipped on `main`
in PR #160 / commit `243d3a6` before this branch was cut). All three
0.3.217 items are internal default/behavior changes with no
Claudius-owned call site: two are new subagent-orchestration env-var
knobs (nested-spawn depth cap 5→1, concurrent-subagent cap default 20)
that Claudius never overrides today (it layers its own independent
per-session spawn-count budget in `lib/server/limits-store.ts`
instead), and one is a Remote Control reconnect bugfix behind a
CLI-only slash command Claudius explicitly does not implement
(`lib/shared/slash-commands.ts:177`, `handler: "native"`). Net: no
product code changes this release. Headline risk to flag: the
subagent nested-spawn depth default silently dropped from 5 to 1
upstream — if any Claudius flow relies on a subagent itself spawning
a further subagent, it will now stop one level earlier with no
Claudius-side override available. See Risks/follow-ups.

## SDK changelog highlights

**0.3.217 (new to this PR):**
- `[skipped — no Claudius call site]` Subagents no longer spawn nested
  subagents by default (depth cap 5→1; override via
  `CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH`). Audited `lib/server/session.ts`
  (`envOverride`/`buildEnvForProfile`, lines ~2011-2075 and ~4478): the
  only env vars Claudius injects into `query()` are per-profile
  auth/proxy vars from `buildEnvForProfile`; there is no settings
  surface for raw SDK env-var tuning and no evidence Claudius's own
  custom subagents (`lib/server/db-agents.ts`) rely on >1 level of
  nesting. This is a real upstream *behavior* change, not just a type
  change — see Risks/follow-ups.
- `[skipped — no Claudius call site]` Added a cap on concurrently-running
  subagents (default 20, override via `CLAUDE_CODE_MAX_CONCURRENT_SUBAGENTS`).
  Claudius already layers its own orthogonal session-lifetime spawn cap
  (`toolBudgetLimits.maxSubagents` / `toolBudgetUsed.subagents` in
  `lib/server/limits-store.ts:26-28`, enforced `lib/server/session.ts:2528-2547`)
  and never overrides the upstream per-session env vars this replaces
  (`CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION`, see the doc comment at
  `lib/shared/tool-budget.ts:3-4`), so following the same precedent for
  the new concurrency knob is consistent, not a hand-wave. Default 20
  is comfortably above Claudius's own workflow concurrency ceiling
  (`min(16, cpu-2)` per the Workflow tool).
- `[skipped — CLI-only feature, no Claudius implementation]` Fixed
  Remote Control sessions not re-sending pending permission prompts to
  reconnecting clients. Remote Control is exposed in Claudius only as
  a passthrough slash command (`lib/shared/slash-commands.ts:177`,
  `id: "remote-control"`, `handler: "native"`, described as "Enable
  remote control from claude.ai (CLI only)") — Claudius has no
  Remote-Control-specific server/client code of its own to patch.

**0.3.216 (already on `main` before this branch, not part of this diff):**
- `[shipped — pre-existing on main]` `rewindFiles` gained an optional
  `skippedLinks` count. Already fully wired end-to-end in
  `components/chat/RewindFilesButton.tsx` (type + `data-testid`
  `rewind-skipped-links-warning`) and `app/api/sessions/[id]/rewind/route.ts`,
  landed in PR #160 (commit `243d3a6`) prior to this branch being cut
  from `main`. No new work this PR; listed for changelog completeness only.
- `[skipped — feature absent from installed SDK]` Changelog claims a
  `tool_result_meta` sidecar (`non_execution_kind`, `user_feedback`) was
  added to user messages. Verified by grepping the entire installed
  `node_modules/@anthropic-ai/claude-agent-sdk` package (`sdk.d.ts`,
  `sdk.mjs`, `bridge.mjs`, `manifest.json`) for `tool_result_meta`,
  `non_execution_kind`, `user_feedback`, and camelCase variants: zero
  matches anywhere. `SDKUserMessage` (sdk.d.ts) has no such field. This
  looks like an upstream changelog/ship mismatch — flagged under
  Risks/follow-ups rather than implemented against a field that doesn't
  exist yet.
- `[skipped — no Claudius surface, low value]` `SDKResultSuccess` gained
  optional `user_message_uuid` / `request_sent_wall_ms` for cross-host
  request-latency correlation. Claudius runs as a single self-hosted
  instance (or single-process Electron build), not a multi-host fleet,
  so there is no second host to correlate against. Checked
  `components/chat/StatusLine.tsx` and `lib/server/session-recap.ts` —
  neither surfaces any existing latency field (`duration_ms`, `ttft_ms`,
  etc.) today, so wiring only these two new fields through with no
  consumer would be dead code with no user-visible payoff.

## Risks / follow-ups

- **Subagent nested-spawn depth silently dropped 5→1 (0.3.217).** If
  any Claudius flow depends on a subagent itself spawning a further
  subagent (e.g. a custom DB-backed agent definition in
  `lib/server/db-agents.ts` that delegates via the Task tool), it will
  now stop one level earlier than before with no Claudius-side
  override — Claudius doesn't expose
  `CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH` as a setting. Watched for this
  during the e2e gate run (see below); if a spec depending on nested
  subagent spawning regresses, that's the first place to look before
  assuming it's unrelated.
- **Upstream changelog/ship mismatch on `tool_result_meta`.** The
  0.3.216 changelog entry describing a `tool_result_meta` sidecar
  (`non_execution_kind`, `user_feedback`) does not correspond to
  anything in the installed `@anthropic-ai/claude-agent-sdk@0.3.217`
  package (verified via full-package grep, see above). Worth flagging
  to the SDK team; re-audit on the next bump in case it ships later
  under this or a different field name — it's a genuinely useful
  feature (classifying denied/interrupted/cancelled tool calls without
  string-matching) that Claudius should wire up once it actually
  exists.
- Gate results (lint/vitest/build/playwright) are pasted in the PR
  description / final run message; re-check those before merging if
