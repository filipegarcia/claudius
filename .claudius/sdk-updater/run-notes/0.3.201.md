## Summary

SDK 0.3.201 is a pure parity bump: its only changelog line is "Updated to
parity with Claude Code v2.1.201". Verified against the tarballs — every
public type surface the SDK ships (`sdk.d.ts`, `agentSdkTypes.d.ts`,
`bridge.d.ts`, `browser-sdk.d.ts`, `sdk-tools.d.ts`) is **byte-identical**
to 0.3.200. The only deltas are the bundled CLI binary files
(`sdk.mjs`, `browser-sdk.js`, `bridge.mjs`) — which carry the vendored
Claude Code v2.1.201 build — plus the `// Version:` stamp. No public
export, type, or documented behaviour that Claudius consumes changed, so
no Claudius code change was required. Headline risk to flag for review:
none — this is a version-stamp + vendored-binary bump with a zero-diff
public API. The CC-2.1.201 *feature* parity (any new user-facing CLI
surface it introduces) is tracked separately by `scripts/cc-parity/`, not
by this SDK bump, since none of it is reachable through a changed SDK
export here.
