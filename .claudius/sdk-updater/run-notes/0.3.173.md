## Summary

SDK 0.3.173 brings parity with Claude Code v2.1.173. Both upstream items
are SDK-internal: Fable 5 `[1m]` model-name suffixes are now normalised
automatically (the context-1m window is included by default in the binary),
and a spurious sandbox-dependency warning on Windows at startup has been
suppressed. Neither item touches any Claudius call-site. The only
repository deliverable is the `package.json` version bump already applied
by the orchestrator. As a separate housekeeping step, pre-existing e2e
flakiness in `tests/e2e/model-picker.spec.ts` and
`tests/e2e/customizations-drawer.spec.ts` was fixed (timing-only; no
new test logic was added or removed).

## Code changes

- No SDK-driven code changes required. Both Claude Code v2.1.173 items
  are entirely SDK-internal; Claudius's `lib/server/session.ts`
  call-sites, `sdk.d.ts` imports, and UI components are unchanged.
- `tests/e2e/model-picker.spec.ts` — added
  `await expect(trigger).toBeVisible({ timeout: 15_000 })` and
  `await expect(trigger).toHaveAttribute("aria-expanded", "false")`
  before each model-picker-trigger click in the 5 tests that previously
  clicked immediately after `page.goto("/")`. Under parallel Playwright
  worker load, React hydration sometimes completes after the click,
  leaving the event handler unattached and the panel never opening.
  The fix mirrors the stabilisation pattern already used in the
  first (always-passing) test in that spec.
- `tests/e2e/customizations-drawer.spec.ts` — added
  `await expect(drawerBtn).toBeVisible({ timeout: 15_000 })` before
  the drawer-button click in the 3 tests that lacked a prior visibility
  wait, for the same reason.

## Risks / follow-ups

- The Fable `[1m]` normalisation is purely internal to the SDK binary.
  If Claudius ever constructs Fable model IDs with an explicit `[1m]`
  suffix (e.g., from user configuration), the SDK will silently strip
  it rather than routing to a separate model variant — worth keeping in
  mind if Fable context-window selection is ever exposed in the UI.
- The 0.3.173 parity entry is intentionally brief; a deeper diff of
  the SDK binary may surface additional behaviour changes the changelog
  doesn't call out. Nothing new appeared in `sdk.d.ts` export shapes
  between 0.3.172 and 0.3.173.
