## Summary

SDK 0.3.191 is a small additive/fix release. It adds an optional `old_source`
field to `NotebookEditOutput` (enabling notebook-cell diff data in tool results),
extends the rate-limit and model-scoped usage types already shipped in 0.3.190,
and fixes a bug where fast mode reverted to standard after the first turn when
`settingSources` included user/project files. All four items require zero Claudius
code changes: the two type/usage additions were already shipped in the 0.3.190
migration (PR #78), the `old_source` field is not consumed by Claudius (tool
results rendered generically), and the fast-mode fix is entirely in the SDK binary.
No code changes, no new UI surfaces, no new tests. Headline risk: none identified.

## Code changes

## Risks / follow-ups

- `old_source` / notebook inline diffs: the SDK now provides the old cell source
  in `NotebookEditOutput.old_source` for replace/delete edits. Claudius currently
  shows raw JSON for all tool results and doesn't render diffs for any tool (Edit,
  MultiEdit, NotebookEdit). Adding a cell-relative diff renderer to ToolCall.tsx
  (keyed on `name === "NotebookEdit"` with `old_source` present) would be a clean
  UX improvement; schedule for a future iteration, not this migration cycle.
