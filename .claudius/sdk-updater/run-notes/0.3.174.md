## Summary

SDK 0.3.174 widens the set of fallback triggers for which the `system/model_fallback` message is emitted. Previously the message was sent only for `model_not_found` and `permission_denied` triggers; it is now also sent for `overloaded`, `server_error`, and `last_resort`, with the `trigger` field gaining the `server_error` and `last_resort` literal values. In Claudius the only change needed was to widen the local TypeScript cast union in `use-session.ts` from `"overloaded" | "model_not_found"` to include all five trigger values. The pill render path is already trigger-agnostic (label comes from `content`, detail renders `trigger` verbatim), so new trigger strings already displayed correctly at runtime — this is a type-annotation catch-up. No model-identity, model-name, context-window, or pricing changes in this release.
