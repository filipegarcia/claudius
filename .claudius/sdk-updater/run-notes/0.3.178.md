## Summary

SDK 0.3.178 delivers four bug fixes / hardening improvements and one runtime error UX improvement. The most impactful for Claudius are: (1) `SDKPermissionDeniedMessage` now reliably carries a typed `decision_reason_type` discriminator (`safetyCheck`, `asyncAgent`, etc.), enabling proper UI rendering; (2) `SDKInformationalMessage` is now correctly emitted when a `UserPromptSubmit` hook blocks a prompt, closing a silent-hang bug; (3) Remote Control workers now send a `worker_shutting_down` system message so clients can explain graceful exits instead of treating them as dead hosts. We wired all three into `use-session.ts`: the `permission_denied` subtype now populates the already-defined `kind: "permission_denied"` SystemEntry (with the ShieldAlert icon and red tone that were already waiting for it), `informational` subtypes render the `content` field as an info pill (surfaces hook block reasons), and `worker_shutting_down` renders the reason string. The `disallowedTools` MCP fix and the spawn-failure libc message are SDK-internal with no Claudius call-site changes required. No model identity changes in this release.

## SDK changelog highlights

- `[shipped]` **Permission-denied advisory messages carry typed denial reasons** (`safetyCheck`, `asyncAgent`). The `SDKPermissionDeniedMessage` (`type: 'system'`, `subtype: 'permission_denied'`) already existed in the d.ts with `decision_reason_type?: string`, but the field was unreliably populated. Now it reliably carries discriminators like `safetyCheck`, `asyncAgent`, `hook`, `rule`, etc. We wired a dedicated handler in `use-session.ts` that creates a `kind: "permission_denied"` SystemEntry — the kind was already defined in `lib/client/types.ts` and the ShieldAlert icon was already registered in `SystemPill.tsx`, just never populated.

- `[shipped]` **UserPromptSubmit hook block feedback now emitted to the SDK event stream**. The `SDKInformationalMessage` (`type: 'system'`, `subtype: 'informational'`) carries a `content` string and a `level` (`'info' | 'notice' | 'suggestion' | 'warning'`) and was previously falling through to the catch-all `system/informational` info pill (losing the content). Added a dedicated handler in `use-session.ts` that renders `content` as the label so hook block reasons (and other informational banners) are visible to users instead of silently swallowed.

- `[shipped]` **Remote Control workers send `worker_shutting_down` on graceful exit**. The `SDKWorkerShuttingDownMessage` (`type: 'system'`, `subtype: 'worker_shutting_down'`) carries a `reason` string. Previously it fell through to the generic `system/worker_shutting_down` info pill (losing the reason text). Added a dedicated handler that renders "Session ended: \<reason\>" as an info pill.

- `[skipped — SDK-internal error messaging]` **Spawn failures on an existing native binary explain the likely libc mismatch**. This is a runtime improvement to the error message text emitted by the SDK binary when it detects a musl/glibc mismatch. No new API surface, no new types, no call-site in Claudius. Claudius surfaces spawn errors through the existing session error path unchanged.

- `[skipped — SDK-side bug fix, no Claudius call-site change]` **MCP server-level specs (`mcp__server`, `mcp__server__*`) in `disallowedTools` now correctly remove all tools from the named server**. Claudius already passes `disallowedTools` correctly in `lib/server/db-agents.ts` (lines 48–49). The bug was entirely in the SDK's tool-filtering logic. No Claudius change needed; we benefit automatically from the bump.

## Code changes

- `lib/client/use-session.ts` — Added three new system-message subtype handlers in the `msg.type === "system"` branch (inserted before the `isSuppressedSystemEvent` catch-all):
  1. `subtype === 'permission_denied'` → creates a `kind: "permission_denied"` SystemEntry showing the denied tool name and `decision_reason_type` as detail.
  2. `subtype === 'informational'` → creates a `kind: "info"` SystemEntry using the message's `content` field as the label; skips empty content to avoid blank pills.
  3. `subtype === 'worker_shutting_down'` → creates a `kind: "info"` SystemEntry with "Session ended: \<reason\>" label so users understand graceful Remote Control worker exits.

## Risks / follow-ups

- The `informational` message handler drops messages with empty `content` silently. If the SDK ever emits an informational with a non-empty `prevent_continuation: true` but no content, the blocked state won't surface in the UI. Low likelihood; flagging for awareness.
- `worker_shutting_down` is marked in the SDK d.ts as durable (may appear mid-stream on a resumed session). We render it as a normal info pill; a resumed session will show it as historical context, which is acceptable but could look surprising. No action required now — document here for the next cycle.
