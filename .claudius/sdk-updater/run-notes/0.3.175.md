## Summary

SDK 0.3.175 adds a single optional field `enforceAvailableModels?: boolean` to the SDK's `Settings` interface. When `true` and `availableModels` is a non-empty array, the "Default" model alias resolves to the first entry in `availableModels` if the user-tier default model is not in the allowlist — this constrains even the implicit default selection, not just the explicit model picker. The field is described as "typically set in managed settings by enterprise administrators," placing it in the same category as `availableModels`, `modelOverrides`, and other enterprise-only keys that Claudius deliberately omits from its settings UI. No code changes were needed: the `ClaudeSettings` type's `[key: string]: unknown` catch-all passes the field through unmodified, and the SDK binary picks it up directly from filesystem settings. No model-identity, model-name, context-window, or pricing changes in this release.
